# 디자인 핸드오프 vs 기능 명세서 — 화면별 차이점

| 항목 | 내용 |
| --- | --- |
| 기준 디자인 | `최종 1차내부 UT 프로토타입.svg`, `컴포넌트.svg` (Downloads) |
| 기준 명세 | [developer-feature-spec.html](./developer-feature-spec.html) |
| 작성일 | 2026-06-06 |

디자인 핸드오프(UT 프로토타입)와 개발자 기능 명세서를 화면별로 대조한 문서입니다. **플로우·IA는 대체로 일치**하며, 차이는 주로 **비주얼·카피·명세서에만 정의된 동작 규칙**에 있습니다.

---

## 요약

| 구분 | 디자인 핸드오프 | 기능 명세서 |
| --- | --- | --- |
| 역할 | 시각·컴포넌트·레이아웃 | 동작·API·엣지케이스 |
| 화면 ID | 없음 (Figma 프레임명) | `SCR-*` 코드 |
| 비주얼 SSOT | **골드 CTA, 오렌지 칩, 그린 확인, Paperlogy, 일러스트** | 보라 와이어프레임, 이모지 목업 |
| 동작 SSOT | 없음 | **타이머, 마이크 상태머신, OX 분기, API** |

**구현 원칙:** UI는 디자인 핸드오프, 로직은 기능 명세서(+ [api-and-database-modeling-plan.md](../../plan/api-and-database-modeling-plan.md))를 따른다.

## 0.1 단원 선택 (`SCR-OB01`)

### 같은 점

- 8개 커리큘럼 칩, 단일 선택, `다음으로` CTA, 진단 버튼 존재

### 다른 점

| 구분 | 디자인 | 명세서 |
| --- | --- | --- |
| 단계 표기 | `1단계: 커리큘럼 선택` | `1 / 2` |
| 진단 버튼 | `실력 진단` | `📝 나의 수학 실력 진단받기` |
| 구분선 카피 | (디자인에 `또는 직접 선택하기` 유무 불명확) | `또는 직접 선택하기` 명시 |
| 기본 선택 | 시각적으로 칩 선택 상태 표현 | 기본값 `분수의 계산` 명시 |
| CTA 스타일 | 골드 그라데이션 버튼 | 보라(`#6c5ce7`) 와이어프레임 |
| 칩 선택색 | 오렌지 (`#FF9438`) | 보라 |
| 동작 규칙 | 없음 | 칩 미선택 시 토스트, `GET /curriculum`, `POST /user/onboarding` 등 |

## 1.0 홈 (`SCR-HOME`)

### 같은 점

- 레벨+코인+설정 헤더, 진행률 바, 캐릭터+말풍선, 3탭 네비, 과외 CTA

### 다른 점

| 구분 | 디자인 | 명세서 |
| --- | --- | --- |
| 배경 | 일러스트(언덕·잔디·하늘) | 단색 + 이모지 캐릭터 |
| 캐릭터 | 일러스트 AI 학생 | 🧒 이모지 |
| 진행률 바 | 그린 | 보라 그라데이션 |
| 탭 활성 | 오렌지 언더라인 | 보라 텍스트 |
| 과외 CTA | 골드 `과외하러가기` | `📚 과외하러가기` |
| 상태 수 | 여러 변형(기본·팝업·수업 후 등) 시각 표현 | State A/B/C 3종 + 동작 정의 |
| **동작** | 2초 타이머 등 **표현 없음** | **2초 후** 과외 팝업 자동 노출, 캐릭터 터치 시 즉시 팝업, `lessonCompletedToday`에 따라 팝업 미노출 |

## 2.0 준비 (`SCR-PREP`)

### 같은 점

- `◀ 돌아가기`, 오늘 수업 주제, `수업 시작하기` CTA

### 다른 점

| 구분 | 디자인 | 명세서 |
| --- | --- | --- |
| GNB/헤더 | 수업 화면 계열은 `내 수업공간` | `◀ 돌아가기`만 (단원명 GNB 없음) |
| 주제 표시 | 카드형 UI | `오늘의 수업` 배지 + `📐 분수의 세계` + 토픽 2줄 목록 |
| 안내 문구 | (디자인에 5초 안내 노출 여부 불명) | `💡 설명할 내용을 머릿속으로 정리해 보세요! (약 5초)` |
| 버튼 상태 | 활성 골드 버튼 위주 | **5초 카운트다운** `(5)→(1)` 후 활성, 그 전 `disabled` |
| CTA 아이콘 | 없음 | `🎓 수업 시작하기` |
| 뒤로가기 | confirm 팝업 시각 있음 | **confirm 없이** 즉시 홈 복귀 |

## 3.0 수업 시작 (`SCR-LESSON`)

### 같은 점

- GNB, AI 질문 말풍선, 마이크, 웨이브폼, 힌트노트(퀘스트북), `설명완료` 버튼

### 다른 점

| 구분 | 디자인 | 명세서 |
| --- | --- | --- |
| GNB 타이틀 | `내 수업공간` | 단원/세션 제목 (예: 분수 관련) |
| 힌트노트 | 갈색 퀘스트북 탭 UI | `📖 힌트 노트 펼치기` 텍스트 탭 |
| 말풍선 | AI=흰색+네이비, 유저=노란색 (컴포넌트 시트) | 와이어프레임 말풍선 |
| `설명완료` | 골드/그린 혼용 가능 | 그린 `✅ 설명완료` |
| **마이크 규칙** | 녹음 중/후 상태만 시각 표현 | **1탭=시작, 2탭=종료** 후에만 `설명완료` 노출 (상태머신 상세) |
| 이탈 | confirm 팝업 있음 | `confirm("수업을 중단하고...")` 문구 정의 |
| 하단 탭 | **없음** (수업 중) | 동일 (숨김) |

## 3.0 AI 반응 (`SCR-REACTION`)

### 같은 점

- AI 오답 말풍선, OX 2버튼, 힌트노트 자동 펼침, 재설명 마이크, `설명 종료`

### 다른 점

| 구분 | 디자인 | 명세서 |
| --- | --- | --- |
| OX 버튼 | `틀렸어!` / `맞았어!` | `❌ 틀렸어!` / `⭕ 정답이야!` |
| Result 말풍선 | 그린 결과 말풍선 컴포넌트 있음 | 상태별 텍스트만 기술 |
| `설명 종료` | 그린 CTA | `🎉 설명 종료` |
| **동작** | 분기 화면만 나열 | **정답 클릭 차단**(`correctBlocked`), 형광펜 300ms 순차, 1.5초 후 OX 숨김, 재설명 **1탭=완료** 등 상세 |
| API 타이밍 | 없음 | 명세는 여기서 `complete` 호출; API 계획서는 `advance-phase`와 분리 |

## 공통·컴포넌트·팝업

### 디자인 `컴포넌트.svg`에만 상세 정의된 항목

| 컴포넌트 | 스펙 |
| --- | --- |
| CTA 버튼 | 골드 그라데이션 (`#FFD700`) |
| Confirm 버튼 | 그린 (`#49CA34`) |
| Secondary 버튼 | 흰 배경 + 네이비 테두리 |
| Disabled 버튼 | 회색 |
| 칩 (선택/미선택) | 오렌지 / 회색 |
| 말풍선 | AI(흰+네이비), 유저(노란), 결과(그린), 힌트(회색) |
| 퀘스트북 | 갈색 책 탭 형태 |
| 탭바 | 홈 / 수업공간 / 수업기록, 활성=오렌지 |
| 폰트 | Paperlogy 8~32px |

### 명세서에만 정의된 항목

| 항목 | 내용 |
| --- | --- |
| 화면 전환 | fade-out 400ms → fade-in 500ms |
| TTS 시뮬레이션 | 마이크 2탭 상태머신 |
| 힌트노트 | `.open`, `.glowing`, `.highlighted` 클래스 동작 |
| 코인/배지 | +500c, 레벨업 조건 |
| API | 화면별 엔드포인트·에러·엣지케이스 |
| `SCR-*` ID | Figma·코드·API 문서 연결 키 |

### 팝업

| 항목 | 디자인 | 명세서 |
| --- | --- | --- |
| 이탈 확인 | 별도 컴포넌트로 시각 정의 | 수업 화면에서만 `confirm` 문구 정의 |
| 축하/보상 팝업 | 일러스트 기반 | 이모지·애니메이션 스펙 |

## 후속 작업 제안

1. 스플래시 → `SCR-SPLASH` 또는 `APP-ENTRY` 하위 화면으로 명세 추가
2. Figma 프레임에 `SCR-*` 이름 부여 (API 계획 §1과 연결)
3. 카피 확정: OX 버튼, GNB 타이틀, 진단 버튼, 설정 저장 라벨
4. 디자인 토큰을 코드용 CSS/테마 변수로 문서화
