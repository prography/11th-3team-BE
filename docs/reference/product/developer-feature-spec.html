1. 화면 전환 규칙

  • 모든 화면 전환은 fade-out(400ms) → fade-in(500ms) 순서로 처리
  • 전환 중 중복 클릭 방지 처리 필수 (버튼 disable 또는 flag)
  • 뒤로가기 시에도 동일한 트랜지션 적용
  • 딥링크/푸시알림으로 특정 화면 직접 진입 시 온보딩 완료 여부 체크 필수

2. 하단 네비게이션 바 표시 규칙

  • 표시 화면: 홈(1.0), 수업기록(7.0)
  • 숨김 화면: 온보딩, 준비화면, 수업공간(Core Loop), 칭찬, 보상, 설정
  • 탭 구성: 홈 / 수업공간 / 수업기록 (3탭)
  • 수업공간 탭 클릭 시 → 2.0 준비 화면으로 직접 이동
  • 현재 활성 탭은 .active — 오렌지(#FF9438) + 하단 언더라인

3. 코인 / 배지 / 레벨 시스템

  • 코인: 세션 1회 완료 시 +500c 지급 (추후 단원 난이도별 차등)
  • 배지 레벨: 세션 완료 횟수 기반 자동 레벨업
  • Lv.1 새내기 선생님 → Lv.2 똑똑한 선생님 → ... (단계별 정의 필요)
  • 코인/레벨은 보상 화면에서 갱신 후, 홈 복귀 시 반영
  • 코인 잔액은 home-header에 항상 노출

4. 퀘스트북 힌트노트 공통 동작

  • 수업공간(3.0) 화면에서만 노출, 하단 펼침 패널 (bottom sheet)
  • 닫힌 상태: 퀘스트북 탭(갈색 책) — 라벨 "힌트노트"
  • 열린 상태: .open 클래스 → bottom: 0 전환 (하단에서 위로 슬라이드)
  • 자동 펼침 트리거: 오답 선택 시 or 정답 오선택 차단 시
  • 글로우 이펙트: .glowing 클래스 → 하단 탭 버튼 발광 애니메이션
  • 형광펜: .highlighted 클래스 → 300ms 딜레이로 순차 적용

5. TTS 시뮬레이션 규칙 (프로토타입)

  • 실제 STT 미구현 — 마이크 UI + Waveform 애니메이션으로 시뮬레이션
  • 마이크 클릭 1회: 녹음 시작 (Waveform 활성 + 힌트노트 탭 글로우)
  • 마이크 클릭 2회: 녹음 종료 (Waveform 비활성 + "설명완료" 버튼 노출)
  • 녹음 중 다른 화면 이동 불가
  • 향후 실제 STT 적용 시 마이크 상태를 API 호출로 대체

6. 커리큘럼 칩 선택 규칙

  • 단일 선택만 허용 (선택 시 나머지 deselect)
  • "실력 진단" 버튼 클릭 시 → 모든 칩 deselect (추후 진단 플로우 연결)
  • 칩 목록: 정수와 유리수 / 일차방정식 / 분수의 계산 / 도형의 기초 / 비례와 반비례 / 통계의 기초 / 소수의 나눗셈 / 규칙과 대응
  • 선택된 칩은 온보딩 ↔ 설정 화면 간 동기화 필수

Description

  • 빈도 선택 토글 TOGGLE

    "주 2회" / "주 3회" 택1. 기본값: 주 3회

    • 선택 변경 시 → 선택된 요일 수가 빈도와 불일치하면 초과 요일 자동 해제 (앞에서부터)
    • 주 2회 선택 상태에서 이미 3개 요일 선택 → 가장 먼저 선택된 요일 1개 해제
  • 요일 선택 버튼 MULTI SELECT

    월~일 7개 버튼 다중 선택. 빈도 수에 맞춰 선택 제한.

    • "주 2회" 선택 시 최대 2개 요일까지만 선택 가능
    • "주 3회" 선택 시 최대 3개 요일까지만 선택 가능
    • 초과 선택 시 → 토스트: "주 N회 설정이에요. 요일을 N개만 골라주세요"
    • 기본값: 화, 목, 토
  • 시간 선택 드롭다운 DROPDOWN

    오후 3시 ~ 오후 8시 (1시간 단위, 6개 옵션). 기본값: 오후 5시

    설정한 요일 + 시간에 과외 시작 푸시알림을 발송하기 위한 설정

  • [설정 완료 및 홈으로 가기 🏠] 버튼 NAV

    클릭 시: 온보딩 데이터 저장 + 푸시알림 스케줄 등록 → 1.0 홈 화면으로 전환

    → 온보딩 완료 플래그를 서버/로컬에 저장하여 재노출 방지

⚠️ Edge Cases

• 요일 미선택 상태에서 [설정 완료] 클릭 → 버튼 비활성 유지 + "요일을 골라주세요" 토스트
• 빈도를 주 3회 → 주 2회로 변경했을 때 이미 3개 요일 선택 → 가장 먼저 선택된 요일 1개 자동 해제
• 푸시알림 권한 미허용 상태에서 [설정 완료] 클릭 → 알림 권한 요청 팝업 노출, 거부 시에도 온보딩은 완료 처리 (알림 없이 사용 가능)
• 뒤로가기(0.1로) 시 → 시간표 설정값 임시 보존, 다시 돌아오면 복원

Description

  • 코인 폭발 이펙트

    디자인 핸드오프: 학부모 일러스트 + 봉투(과외비) + +500 코인 강조. 선택적으로 코인 방사형 팡파레 애니메이션.

    • 배경: 밝은 웜톤 일러스트 (ph-reward-bg)
    • 코인 burst 구현 시: CSS --tx, --ty 벡터, 0.8s 1회성
  • 보상 카드

    코인 스핀 아이콘 + "+500c" 카운트업 애니메이션 + "과외비 입금!" 라벨

    • 카운트업: 0 → 500 (25씩 증가, 30ms 간격, 약 0.6초 소요)
    • 학부모 캐릭터 + 감사 말풍선: "우리 아이를 가르쳐줘서 감사합니다!"
    • 보상 금액은 서버 응답 기반 (추후 난이도별 차등)
  • 배지 레벨업 팝업

    🎖️ "'똑똑한 선생님' 배지 레벨업!" + 회전 글로우 이펙트

    레벨업 조건 충족 시에만 노출 (매 세션 노출은 아님)

  • [확인] 버튼 NAV Confirm Green

    클릭 시 → 1.0 홈 (State C)으로 복귀

    • 복귀 시 갱신: 코인 잔액, 레벨명, 진척도, 말풍선, 수업기록 카드
    • 보상 화면 → 홈 전환은 특별 로직 (home-return) 사용
    • lessonCompleted = true 플래그 세팅
⚠️ Edge Cases

• GET /api/session/{sessionId}/reward 실패 시 → 토스트 + 재시도
• 배지 레벨업 조건 미충족 시 → 배지 팝업 미노출, 코인만 표시
• [확인] 중복 클릭 방지 → 즉시 disabled 후 POST /reward/ack
• 앱 재진입: pendingRewardSessionId 있으면 SCR-REWARD mount 시 GET /reward만 호출
• 참고: POST /session/complete는 SCR-REACTION [설명 종료]에서 이미 호출됨

    온보딩(0.2)과 동일한 빈도/요일/시간 선택 UI (푸시알림 발송 스케줄 변경)

    현재 설정값이 사전 선택된 상태로 로드. 유효성 검증 규칙도 0.2와 동일

  • [저장하기 ✅] 버튼 ACTION

    클릭 시 → 변경사항 서버 저장 → 1.0 홈으로 복귀

    • 저장 실패: 토스트 "저장에 실패했습니다. 다시 시도해주세요"
